Workspace Rules (P1 - StudySpot)

These rules define the coding workflow for the StudySpot project during Prototype 1 (Scaffolding phase).

---

Naming Conventions
- Components: PascalCase (e.g., LocationCard.jsx)
- Variables: camelCase (e.g., locationList)
- Files: camelCase
- API routes: lowercase (e.g., /locations)

---

Commit Message Guidelines
- Use short, clear, and meaningful commit messages
- Format: [scope]: description

Examples:
- client: initialize React app
- api: create Express server
- docs: add PRD and task list

---

Branching Strategy (P1)
- Single branch workflow using **main**
- All work is committed directly to main
- Optional: short-lived feature branches (e.g., feature/filter-api), then merged back into main

---

Pull Request & Review Process (P1)
- Pull requests are optional for Prototype 1
- If used:
  - Include a short description of changes
  - Reference the related task
- Code should be reviewed before merging when working in a team
